# sekigae-maker

# 席替えアプリ

毎月の席替えを楽にするためのWebアプリ。出席番号をもとに、配置条件を考慮しながらランダムに席替え表を生成します。

## 特長

- ✅ ブラウザだけで動作（インストール不要）
- ✅ 配置条件を画面上で自由に編集可能
- ✅ 名簿を登録すれば席に名前も表示
- ✅ 設定・名簿はJSONファイルで保存・読み込み可能
- ✅ そのまま印刷OK
- ✅ スマホ・タブレット・PC対応

## 使い方

### ローカルで使う場合

1. このリポジトリをダウンロードまたはクローン
2. `index.html` をブラウザで開く

### Web公開する場合

後述の「デプロイ方法」を参照。

## 仕様

### レイアウト

- 34人クラス用
- 縦2列の塊×横3つ（左ブロック5行、中央ブロック6行、右ブロック6行）
- 通路を中央2箇所に配置
- 合計34席（空席なし）

### 配置条件（画面上で編集可能）

| 条件 | 説明 |
|---|---|
| 近くにしたい番号グループ | 同じグループに登録した番号同士をできるだけ近くに配置（最大4グループまで色分け表示） |
| 離したい番号ペア | 登録した番号同士をできるだけ離れた席に配置 |
| 前方に配置したい番号 | 1〜2列目を優先して配置 |

### 配置アルゴリズム

1. 200回のランダム試行を実行
2. 各配置を以下の評価関数でスコアリング：
   - 離したいペアの距離が遠いほど加点（距離3未満は大幅減点）
   - 近くにしたいグループ内の総距離が近いほど加点
3. 最高スコアの配置を採用

### 距離の定義

ブロック内のマンハッタン距離 + ブロック間の差 × 4（ブロックをまたぐ場合は重めにペナルティ）。

## カスタマイズ

クラスの人数や教室レイアウトを変えたい場合は、`script.js` の以下の定数を編集：

```javascript
const TOTAL = 34;        // クラスの人数
const BR = [5, 6, 6];    // 左・中・右ブロックの行数
```

合計人数 = `BR[0]*2 + BR[1]*2 + BR[2]*2` になるように設定してください。

## デプロイ方法

### GitHub Pages（無料・最も手軽・推奨）

1. GitHubでリポジトリを作成し、このフォルダの中身をプッシュ
2. リポジトリの **Settings → Pages** へ
3. **Source** で `main` ブランチの `/ (root)` を選択
4. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` で公開される

### Vercel（無料・自動デプロイ）

1. [Vercel](https://vercel.com/) にGitHubアカウントでログイン
2. 「New Project」から該当リポジトリを選択
3. デフォルト設定のまま「Deploy」
4. 数十秒で公開URLが発行される

### Netlify（無料・GitHub不要でもOK）

1. [Netlify](https://www.netlify.com/) にログイン
2. ZIP形式のフォルダをドラッグ&ドロップでアップロード
3. すぐに公開URLが発行される

### Cloudflare Pages（無料・高速）

1. [Cloudflare Pages](https://pages.cloudflare.com/) にログイン
2. GitHubリポジトリを連携
3. ビルド設定なしでデプロイ

> **どれを選ぶ？**
> - GitHubで管理するなら → **GitHub Pages** が最もシンプル
> - 自動デプロイや独自ドメインを使いたい → **Vercel** または **Cloudflare Pages**
> - GitHubを使わずに公開したい → **Netlify** のドラッグ&ドロップが手軽
